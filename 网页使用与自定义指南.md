# 个人网页使用与自定义指南

这份手册说明如何修改个人信息、简历、技能、笔记、照片、主题和联系方式，以及如何把更新发布到 GitHub Pages。

## 一、网站文件结构

```text
Drizzle259.github.io/
├── index.html          # 网页结构，一般不需要修改
├── style.css           # 颜色、尺寸、排版和响应式样式
├── script.js           # 动效、笔记读取、主题切换等交互
├── config.js           # 最常修改：个人信息、履历、技能和笔记目录
├── notes/              # Markdown 学习笔记
├── assets/             # 可自行创建，用来放照片、简历和图片
├── README.md
└── 网页使用与自定义指南.md
```

日常更新优先修改 `config.js` 和 `notes/`，通常不用改 `index.html` 与 `script.js`。

当前 `assets/` 中还包含三个表情贴纸和一张经过网页虚化、降饱和处理的背景图。替换同名文件即可更换素材，不必改代码：

```text
assets/sticker-love.png
assets/sticker-star.png
assets/sticker-sparkle.png
assets/background-art.jpg
```

公开发布前，请确认图片允许用于公开网页，并按原作者要求署名。

## 二、修改个人信息

打开 `config.js`，找到 `profile`：

```js
profile: {
  fullName: "你的姓名或网名",
  firstName: "首页第一行",
  lastName: "首页第二行",
  initial: "头像缩写",
  location: "所在城市",
  photo: "assets/avatar.jpg",
  roles: ["学生", "Python 学习者", "知识记录者"],
  intro: "首页的一句话介绍",
  aboutLead: "关于我区域的醒目文字",
  aboutDetail: "更完整的自我介绍",
  motto: "你的座右铭",
  contactText: "联系区域的说明"
}
```

`roles` 会在首页自动轮换显示，可以自由增加或删除。

## 三、添加个人照片

1. 在网站根目录创建 `assets` 文件夹。
2. 把照片放进去，例如 `assets/avatar.jpg`。
3. 修改 `config.js`：

```js
photo: "assets/avatar.jpg"
```

推荐使用正方形或竖版照片，文件尽量小于 1 MB。留空 `photo: ""` 时，网页会显示字母头像。

## 四、修改个人简历和经历

在 `config.js` 找到 `timeline`。每一对花括号代表一段经历：

```js
{
  period: "2024 - 2028",
  type: "教育经历",
  title: "计算机科学与技术",
  place: "某某大学",
  description: "主修课程、成绩、奖项或学习重点。"
}
```

可以复制这一整段来增加经历，也可以删除不需要的项目。经历可以是：

- 教育经历
- 实习或工作
- 课程项目
- 比赛与奖项
- 志愿活动
- 阶段性学习目标

网页中的“保存简历”会调用浏览器打印功能。选择“另存为 PDF”即可得到简历 PDF。

## 五、修改技能

在 `config.js` 找到 `skills`：

```js
{ name: "Python", level: 78 }
```

- `name` 是技能名称。
- `level` 是 0 到 100 的展示进度。
- 首页会显示全部技能标签，履历区域展示前五项技能进度。

## 六、修改联系方式

找到 `social`：

```js
social: [
  { label: "GitHub", url: "https://github.com/你的用户名", icon: "github" },
  { label: "Email", url: "mailto:你的邮箱", icon: "mail" }
]
```

当前示例邮箱 `your-email@example.com` 必须换成你的真实邮箱；如果不想公开邮箱，可以删除对应整行。

还可以增加其他链接，例如：

```js
{ label: "Bilibili", url: "https://space.bilibili.com/你的ID", icon: "video" }
```

图标名称来自 Lucide 图标库：<https://lucide.dev/icons/>

## 七、添加一篇公开学习笔记

### 第一步：创建 Markdown 文件

在 `notes/` 中新建文件，例如：

```text
notes/python-list.md
```

内容示例：

````markdown
# Python 列表学习笔记

## 列表是什么

列表是一种有序、可修改的数据结构。

```python
numbers = [1, 2, 3]
numbers.append(4)
```
````

### 第二步：在 config.js 登记

将下面内容添加到 `notes` 数组中：

```js
{
  title: "Python 列表学习笔记",
  summary: "整理列表的创建、访问和常用方法。",
  category: "Python",
  date: "2026-07-12",
  readTime: "5 分钟",
  color: "#ec6b4f",
  file: "notes/python-list.md"
}
```

注意各项之间要保留英文逗号。`file` 必须与真实文件路径完全一致，包括大小写。

## 八、修改配色与背景

网页右上角调色板按钮可以切换四种主题，也能上传当前浏览器使用的背景图片。这类设置只保存在当前设备。

要永久修改所有访客看到的颜色，在 `config.js` 修改 `themes`：

```js
{
  name: "我的主题",
  accent: "#ec6b4f",   // 强调色
  accent2: "#436f65",  // 辅助色
  bg: "#f5f2eb",       // 页面背景
  ink: "#171a19"       // 正文颜色
}
```

默认使用第一套主题。可使用 <https://coolors.co/> 或 <https://colorhunt.co/> 挑选配色。

如果要永久使用背景图片，可以在 `style.css` 中找到 `.site-bg`，添加：

```css
background-image: url("assets/background.jpg");
background-size: cover;
background-position: center;
```

目前默认背景就是 `assets/background-art.jpg`，网页通过 CSS 自动进行模糊、降饱和及低透明度处理，以免干扰文字阅读。

## 九、记录网站更新与笔记发布

主页的“创作与更新”日历从 `config.js` 的 `activity` 数组读取记录。每次修改网站或发布笔记后，增加一条：

```js
{
  date: "2026-07-12",
  type: "website",
  title: "调整首页布局",
  detail: "优化手机端排版和导航。"
}
```

`type` 支持：

- `website`：网页修改，显示魔法棒图标；
- `note`：笔记发布，显示文档图标。

笔记发布时，可以同时在 `notes` 和 `activity` 中各增加一项。日历会自动在对应日期显示圆点，点击日期即可筛选当天记录。

## 十、网站使用的美化工具

- **AOS**：滚动进入动画。
- **Lucide Icons**：轻量线性图标。
- **VanillaTilt**：头像和笔记卡片的轻微立体倾斜。
- **Google Fonts**：Playfair Display、DM Sans 和 Noto Sans SC 字体。
- **CSS 自定义属性**：集中控制配色，并支持主题切换。

这些工具通过 CDN 加载。即使其中某个 CDN 暂时不可用，网站文字、布局、笔记和导航仍可使用，只会少部分动画或图标。

Hexo、MkDocs、VitePress 和 Jekyll 属于不同的静态网站生成器，不能同时套在一个网页中。本网页保持纯 HTML/CSS/JavaScript，优点是结构直观、无需构建、可直接部署到 GitHub Pages。

## 十一、本地预览网页

不能直接双击 `index.html` 测试笔记，因为浏览器会限制读取 Markdown 文件。

在 PyCharm 终端进入网站目录，运行：

```powershell
python -m http.server 8000
```

然后访问：

```text
http://localhost:8000
```

结束预览时，在终端按 `Ctrl + C`。

## 十二、发布更新到 GitHub Pages

修改完成后，在项目目录执行：

```powershell
git add .
git commit -m "Update personal website"
git push origin main
```

GitHub Pages 通常会在几分钟内自动更新。网页地址：

```text
https://drizzle259.github.io/
```

如果没有立即更新，可以按 `Ctrl + F5` 强制刷新，或检查仓库的 `Actions` 页面。

## 十三、常见问题

### 笔记卡片能看到，但点击后无法打开

检查 `config.js` 中的 `file` 路径和实际文件名是否一致。本地必须通过 `python -m http.server 8000` 预览。

### 修改后网页没有变化

确认已提交并推送到 `main` 分支，等待 GitHub Pages 部署完成，再强制刷新。

### 中文文件名是否可以使用

可以，但图片和 Markdown 文件更推荐使用小写英文及连字符，例如 `python-basics.md`，可以减少路径和链接问题。

### 如何删除某个区域

可以在 `index.html` 中删除对应的 `<section>...</section>`，但修改前建议先复制一份文件备份。
